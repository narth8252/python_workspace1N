## 1. 순차 검색(Sequential Search, 순서대로 검색)
- 데이터가 저장된 순서대로 1, 2, 3, 4, ... 순서로 처음부터 하나씩 데이터확인하는 탐색방법
- 데이터가 n개 있을 때 최악 사례로 찾으려는 데이터가 마지막 위치에 있으면 \(O(n)\)의 시간복잡도를 갖습니다.
- 즉, 데이터 수가 많아질수록 탐색 시간이 선형적으로 늘어나 속도가 느립니다.
- 오라클(Oracle) 같은 DBMS에서는 순차 검색을 하면서 병렬 처리로 여러 데이터를 동시에 읽어 성능을 향상
- MySQL 또한 병렬처리 되며, 성능개선을 시도.

## 2. 색인 순차 검색(Index Sequential Search)
- 색인(Index)은 도서 뒤쪽에 있는 목차 같은 기능으로, 원하는 데이터가 어느 구간에 있는지 빠르게 찾도록 도와줍니다[1].
- 데이터가 100만 건이라고 할 때, "AA(1~1000)" 식으로 데이터가 구간별로 정리되어 있다고 가정하면, 
  예를 들어 A 구간이 1~100번, B 구간이 101~200번, C 구간이 201~300번 이런 식입니다.
- 색인 순차 검색은 먼저 인덱스 테이블에서 탐색할 구간을 찾고, 그 구간 내에서만 순차 탐색을 하는 방식입니다[4][5].
- 인덱스 테이블과 주 데이터 리스트 모두 정렬돼 있어야 하며, 인덱스 범위에 따라 탐색 범위를 좁혀 효율을 높입니다.
- 이 방법은 순차 검색에 비해 훨씬 빠르며, 전체 데이터를 일일이 살피지 않아도 되는 장점이 있습니다.

## 3. MySQL과 오라클에서 인덱스의 역할
- MySQL, 오라클 등 대부분의 관계형 DBMS는 인덱스를 사용해 데이터를 빠르게 찾도록 최적화되어 있습니다.
- 인덱스는 특정 컬럼 값과 해당 레코드 위치를 쌍으로 만들어 저장하며, 정렬된 상태를 유지해 빠른탐색 가능
- 단, 인덱스가 많아지면 저장 및 수 시 오버헤드 발생하므로 적절히사용.

## 요약
| 탐색 방법      | 특징                                                           | 시간 복잡도              | DBMS 활용                                                         |
|-----------------|---------------------------------------------------------|--------------------------|---------------------------------------------------------------|
| 순차검색       | 데이터 처음부터 끝까지 순서대로 검색             | 최악의 경우 \(O(n)\) | 단순하지만 큰 데이터는 느림                                 |
| 색인순차검색 | 인덱스 테이블에서 범위 먼저 찾고 범위 내 탐색 | \(O(\log m) + O(n/m)\) (m=인덱스 크기) | 인덱스 활용으로 탐색 속도 향상 |

즉, 색인 순차 검색은 인덱스 테이블에서 구간을 찾고 해당 구간의 데이터만 순차 검색하여 전체 탐색 시간을 줄이는 효과적인 방법입니다.
